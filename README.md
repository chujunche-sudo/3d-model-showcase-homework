# 3D模型展示作业

这是一个围绕“家乡文化服装设计展示”的课程作业项目。项目目标不是完成工业级服装建模，而是用 AI 设计、3D 模型展示、演示页面和 PPT 组合出一个完整的课堂展示方案。

## 项目目标

- 结合家乡历史文化，设计一套服装概念方案
- 生成服装效果图和展示素材
- 用可交互的 3D 模型页面完成演示
- 最终服务于课堂展示，而不是商业上线

## 项目内容

- `index.html`
  3D 模型展示页面
- `model-viewer.min.js`
  本地离线使用的 3D 查看库
- `model.glb`
  用于演示的 3D 模型
- `start-demo.bat`
  一键启动离线演示

## 项目流程

### 1. 明确作业要求

先把作业拆分成几个部分：

- 文化背景
- 服装设计理念
- 展示素材
- 课堂讲解

本项目最终采用的是“概念设计 + 演示模型 + 视频/PPT”组合方式。

### 2. 确定服装设计方向

可以先根据家乡文化整理关键词，例如：

- 历史背景
- 地域色彩
- 代表性纹样
- 建筑或自然景观

再用这些关键词生成服装设计概念图。

### 3. 获取 3D 模型

有两种常见方式：

#### 方式 A：自己建模

- 使用 Blender 等软件建模
- 优点是更贴合自己的设计
- 缺点是学习成本高，时间消耗大

#### 方式 B：下载现成模型再用于展示

- 可以在 Sketchfab 等网站寻找接近自己设计风格的模型
- 下载格式优先选择 `.glb`
- 更适合作业展示和快速落地

本项目实际采用的是这一种方式，因为它更稳定、完成度更高。

### 4. 搭建离线展示页面

使用本地 HTML 页面和 `model-viewer` 展示 `.glb` 模型，实现：

- 鼠标拖动旋转
- 滚轮缩放
- 自动旋转

为了适应课堂环境，页面按“离线可运行”来设计。

### 5. 演示方案优化

在实际操作中，3D 页面经常会遇到：

- 浏览器本地文件权限限制
- 模型太大导致加载缓慢
- 学校电脑性能不稳定

所以最终展示建议准备两套内容：

- 主展示：可交互 3D 页面
- 备用展示：录屏视频或 PPT 页面

## 我在项目中的真实分工

为了如实记录这个项目，建议明确区分 AI 和人工工作。

### AI 参与部分

- 帮助梳理作业思路
- 辅助生成服装设计提示词
- 协助分析展示技术路线
- 协助生成和修改网页代码
- 帮助定位黑屏、离线打开失败、模型过大等问题

### 我自己完成的部分

- 确定作业方向和展示目标
- 挑选合适的模型资源
- 下载模型和整理文件
- 实际测试离线展示是否可运行
- 根据课堂场景决定最终演示方式

## 遇到的问题

### 1. 网页黑屏

原因通常包括：

- 直接双击 `index.html`，浏览器对本地资源有限制
- 模型路径或格式不合适
- 模型过大，加载时间过长

### 2. 模型太卡

主要原因是模型和纹理体积过大，不适合普通电脑现场展示。

### 3. “设计图”和“3D模型”不完全一致

这是这类课程作业里很常见的问题。项目重点是表达设计理念和展示能力，而不是完成专业级服装工业建模。

## 使用方法

双击：

`start-demo.bat`

- PPT
- AI 生成的服装概念图

- 本项目离线页面作为现场加分项
